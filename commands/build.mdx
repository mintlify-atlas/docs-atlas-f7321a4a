```bash
faisanbridge build
```

The `build` command processes every `index.html` in `src/pages/`, `src/posts/`, and `src/templates/`, transpiles components and assets, and writes the resulting files to `dist/`.

## What it does

<Steps>
  <Step title="Clean the output folder">
    Removes the entire `dist/` folder before building to ensure a clean output.
  </Step>
  <Step title="Read source files">
    Scans `src/pages/`, `src/posts/`, and `src/templates/` for `<slug>/index.html` files.
  </Step>
  <Step title="Transpile components">
    Replaces `<div data-faisan-component>` elements with the content of the referenced HTML partial, substituting template variables.

    ```html
    <!-- Before build -->
    <div data-faisan-component="hero.html" data-faisan-headline="Welcome"></div>

    <!-- hero.html contains: <section class="hero"><h1>{{headline}}</h1></section> -->

    <!-- After build -->
    <section class="hero"><h1>Welcome</h1></section>
    ```
  </Step>
  <Step title="Inline assets">
    - Replaces `<link data-faisan-asset>` tags with inline `<style>` blocks containing the CSS file's contents.
    - Replaces `<script data-faisan-asset>` tags with inline `<script>` blocks containing the JS file's contents.
    - Preserves the `type="module"` attribute on script tags if present.
  </Step>
  <Step title="Remove dev loader tags">
    Strips any `<script>` tags whose `src` references `dev-loader.js`.
  </Step>
  <Step title="Write to dist/">
    Writes the compiled file to `dist/<type>/<slug>.html`.
  </Step>
</Steps>

## Component transpilation

Components are HTML partials stored in `src/components/`. Reference them with `data-faisan-component` pointing to the component filename. Pass variables using additional `data-faisan-<key>` attributes тАФ each becomes a `{{key}}` placeholder in the component.

```html
<!-- src/pages/home/index.html -->
<div data-faisan-component="hero.html" data-faisan-headline="Welcome" data-faisan-sub-text="Get started today"></div>
```

The component file receives the values via `{{headline}}` and `{{sub_text}}` (hyphens in attribute names are converted to underscores):

```html
<!-- src/components/hero.html -->
<section class="hero">
  <h1>{{headline}}</h1>
  <p>{{sub_text}}</p>
</section>
```

Result after build:

```html
<!-- dist/pages/home.html (after build) -->
<section class="hero">
  <h1>Welcome</h1>
  <p>Get started today</p>
</section>
```

## Asset inlining

Stylesheets and scripts marked with `data-faisan-asset` are replaced with their inline equivalents. Asset paths are resolved relative to the page's source folder (e.g., `src/pages/home/`).

```html
<!-- Before -->
<link rel="stylesheet" href="styles.css" data-faisan-asset>
<script src="app.js" data-faisan-asset></script>

<!-- After -->
<style data-faisan-injected="styles.css">
/* contents of styles.css */
</style>
<script data-faisan-injected="app.js">
/* contents of app.js */
</script>
```

## Example output

```bash

```


<Tip>
  Always run `build` before `push`. The `push` command reads compiled files from `dist/`, not raw source from `src/`.
</Tip>
