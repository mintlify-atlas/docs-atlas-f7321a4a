---
title: status
description: Report files that are pending merge or have unresolved conflicts.
---

```bash
npm run faisanbridge status
```

The `status` command scans your `src/` folders and prints a summary of which items have files waiting to be merged (`index-pending-to-merge.html`) and which have unresolved conflicts (`index-diff.html`).

No network request is made. `status` only reads the local file system.

## What it reports

| Condition | File present | Meaning |
|-----------|--------------|---------|
| Pending merge | `index-pending-to-merge.html` | Content fetched from WordPress, not yet merged |
| Conflict | `index-diff.html` | Merge produced conflicts requiring manual resolution |

## Example output

```bash
npm run faisanbridge status
```

```
Status:

Pending merge (fetched from WordPress):
  pages/my-page
  templates/home-template

Conflicts (resolve manually):
  posts/hello-world

Total pending: 2
Total conflicts: 1
```

When everything is clean:

```
Status:

No pending merges.
No conflicts.
```

## Next steps based on status

- **Pending merge** — run [`merge`](/commands/merge) to reconcile the fetched content with your local file.
- **Conflict** — open `index-diff.html`, resolve the `<<<<<<< HEAD` / `>>>>>>> REMOTE` markers, save the result as `index.html`, then run `merge` again.
- **Want to discard everything** — run [`abort`](/commands/abort) to remove all pending and conflict files.
