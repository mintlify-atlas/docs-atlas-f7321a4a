---
title: status
description: Report files that are pending merge or have unresolved conflicts.
---

```bash
faisanbridge status
```

The `status` command scans your `src/` folders and prints a summary of which items have files waiting to be merged (`index-pending-to-merge.html`) and which have unresolved conflicts (`index-diff.html`).

No network request is made. `status` only reads the local file system.

## What it reports

| Condition | File present | Meaning |
|-----------|--------------|---------|
| Remote file | `index-pending-to-merge.html` | Content fetched from WordPress, not yet merged |
| Diff file | `index-diff.html` | Merge produced conflicts requiring manual resolution |

## Example output

```bash
faisanbridge status
```

When there are pending files:

```
📦 Status...
⚠️ [pages] my-page: Remote file found.
⚠️ [templates] home-template: Remote file found.
⚠️ [posts] hello-world: Diff file found.
Total items fetched from WordPress: 2
Total items pending to merge: 1
```

When everything is clean (no output lines for individual items):

```
📦 Status...
Total items fetched from WordPress: 0
Total items pending to merge: 0
```

## Next steps based on status

- **Remote file found** — run [`merge`](/commands/merge) to reconcile the fetched content with your local file.
- **Diff file found** — open `index-diff.html`, resolve the `<<<<<<< HEAD (Local)` / `>>>>>>> REMOTE (WordPress)` markers, then run `merge` again.
- **Want to discard everything** — run [`abort`](/commands/abort) to remove all pending and conflict files.
