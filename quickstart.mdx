title: "Quickstart"
description: "Configure the CLI, create your first page, and push it to WordPress in minutes."
---

This guide walks you through setting up a Faisan Bridge project from scratch and syncing your first page to WordPress.

<Steps>

  <Step title="Install the CLI">
    Install `faisanbridge` globally:

    ```bash
    npm install -g faisanbridge
    ```

    Verify the installation:

    ```bash
    faisanbridge --help
    ```
  </Step>

  <Step title="Create your .env file">
    In the root of your project, create a `.env` file with your WordPress site URL and Faisan Bridge API credentials:

    ```bash .env
    FAISAN_PUBLIC_KEY=your_public_key
    FAISAN_SECRET_KEY=your_secret_key
    WORDPRESS_URL=https://your-wordpress-site.com
    ```

    <Tip>
      You can find `FAISAN_PUBLIC_KEY` and `FAISAN_SECRET_KEY` in the Faisan Bridge plugin settings inside your WordPress admin dashboard.
    </Tip>

    <Warning>
      Never commit your `.env` file to version control. Add it to your `.gitignore`.
    </Warning>
  </Step>

  <Step title="Create your project structure">
    Create the source directory and add your first page:

    ```bash
    mkdir -p src/pages/my-page
    ```

    Create `src/pages/my-page/index.html` with your page content:

    ```html src/pages/my-page/index.html
    <!DOCTYPE html>
    <html>
      <head>
        <title>My Page</title>
      </head>
      <body>
        <h1>Hello from Faisan Bridge</h1>
        <p>This content is managed locally and synced to WordPress.</p>
      </body>
    </html>
    ```

    The full project structure supports pages, posts, templates, and components:

    ```
    your-project/
    ├── src/
    │   ├── pages/
    │   │   └── my-page/
    │   │       └── index.html
    │   ├── posts/
    │   ├── templates/
    │   └── components/
    ├── assets/
    │   └── images/
    └── .env
    ```
  </Step>

  <Step title="Build your project">
    Compile your source files into the `dist/` output directory:

    ```bash
    faisanbridge build
    ```

    This processes your HTML files and writes built output to `dist/pages/`, `dist/posts/`, and `dist/templates/`.
  </Step>

  <Step title="Push to WordPress">
    Upload the built files to your WordPress site:

    ```bash
    faisanbridge push
    ```

    On the first push, a `faisan-manifest.json` file is created in your project root. This file tracks MD5 hashes of your content so that subsequent pushes only upload changed files.

    <Note>
      To push all files regardless of whether they have changed, use the `--force` flag:

      ```bash
      faisanbridge push --force
      ```
    </Note>
  </Step>

  <Step title="Check sync status">
    Verify the state of your local project against what is live on WordPress:

    ```bash
    faisanbridge status
    ```

    This shows which local files have pending merge files fetched from WordPress or unresolved conflict diff files.
  </Step>

</Steps>
