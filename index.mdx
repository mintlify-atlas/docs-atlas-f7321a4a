## What is Faisan Bridge?

Faisan Bridge CLI is a developer tool that connects your local IDE to a WordPress site running the Faisan Bridge plugin. It lets you author pages, posts, and templates as plain HTML files locally, then sync them to WordPress over a secure, signed API.

## How it works

<Steps>
  <Step title="Author locally">
    Write HTML pages, posts, and templates in `src/pages/`, `src/posts/`, and `src/templates/`. Use reusable components with `data-faisan-component` attributes.
  </Step>
  <Step title="Build">
    Run `faisanbridge build` to compile your source HTML — inlining components and local assets — into the `dist/` folder.
  </Step>
  <Step title="Push to WordPress">
    Run `faisanbridge push` to send your compiled HTML to WordPress via the Faisan Bridge REST API. Only changed files (tracked by MD5 hash) are uploaded.
  </Step>
  <Step title="Fetch and merge">
    Run `faisanbridge fetch` to pull any changes made in WordPress, then `faisanbridge merge` to reconcile differences with your local files.
  </Step>
</Steps>

## Key features

- **Incremental sync** — Only files whose content has changed (tracked by MD5 hash in `faisan-manifest.json`) are pushed, keeping syncs fast.
- **Secure authentication** — Every API request is signed with HMAC-SHA256 using your secret key and a timestamp, preventing replay attacks.
- **Component system** — Use `data-faisan-component` attributes to reference reusable HTML partials that are inlined at build time.
- **Asset handling** — Images referenced in your HTML are automatically uploaded to WordPress and their local paths are replaced with remote CDN URLs.
- **Conflict resolution** — When local and remote HTML differ, the CLI produces a diff file with `<<<<<<< HEAD` / `>>>>>>> REMOTE` markers for manual resolution.
- **Supports pages, posts, and templates** — Sync standard WordPress post types as well as custom `faisan_template` entries.
